<Info>
  **Prerequisite** Você precisa ter Node.js (versão 18.10.0 ou superior), Java versão 21 ou superior e o Docker.
</Info>

Passo 1. Baixe o projeto do front-end, instale as dependências e rode o projeto.

<CodeGroup>

```bash npm
npm i
```

```bash npm
npm run dev
```
</CodeGroup>


O servidor do front-end irá subir na porta 3000, as configurações de variáveis de ambiente ficam no arquivo .env na raiz do projeto, use o arquivo .env.example como base.

Passo 2. Iniciar o backend

O backend funciona com perfis de desenvolvimento, o perfil dev utiliza as dependências do KeyCloak para autenticação e MongoDB para salvar os dados.

Para utilizar uma versão mais simplificada, recomendo rodar o Java com o perfil sandbox. Então, após ter compilado o JAR, rode com o comando:

```bash
--spring.profiles.active=sandbox
```

O backend irá subir na porta 8080 e estará disponível em `http://localhost:8080`.

### Dependencias em containers

Para subir todas as dependências necessárias para o backend, utilize o arquivo docker-compose.yml na raiz do projeto.

```bash
sudo docker-compose up -d
```

Após esses passos, você já estará apto para fazer suas próprias alterações locais.

## Whatsapp Engine (motor)

O módulo do software que faz as regras de conversação com o WhatsApp é um módulo privado. Para facilitar esse processo, o backend conversa com ele somente através de interfaces, então para rodar localmente você precisa implementar essas interfaces.

Essa interface esta em 

```md
src/main/java/com/flowbot/application/http/BotBuilderApi.java
```

## Gestão de arquivos

Atualmente, esse projeto utiliza um serviço de armazenamento de arquivos próprio que é gratuito.
 Caso você tenha interesse em usar o seu próprio, basta alterar a função que faz o upload de arquivo no front-end, localizada em.

```md
src/api/api-defaults.js  -> uploadResource(resourceFile)
```
